citizen
Purpose: manages all user information

complaint
Purpose: stores all civic issues reported by users.

admin
Purpose:view complaint and changes status to resolved once resolved.

category
Purpose: groups complaints under types.(like: 'road issue','street light')

feedback
Purpose: captures citizen satisfaction after resolution.

department
Purpose: represents government departments handling specific issues.

attachment 
Purpose: stores complaint-related media (photos, videos, documents).

response
Purpose: conversation  between user and admin within a complaint.

analytics(dashboard)
Purpose:system generates dashboards for-
number of complaints per category
average resolution time per department
